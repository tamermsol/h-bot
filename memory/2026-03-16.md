## Build 112 (commit `a20041d`)
- Fixed bottom sheet disappearing on iOS in device control screen
- Root cause: MQTT `watchCombinedDeviceState` stream calling `setState()` while `showModalBottomSheet` was open
- Fix: Added `_isBottomSheetOpen` flag to skip setState during sheet display, apply pending updates on close via `.then()`
- Codemagic build: `69b7c526306422dc8cb36dde`
- ASC build ID: `2557c61a-6f06-4567-9efb-2e70e49f9443`

## Build 113 (commit `aac239b`)
- Fixed device name not saving during provisioning
- Root cause: Line 2177 in `add_device_flow_screen.dart` — `_deviceNameController.text = deviceInfo.module` was overwriting user's custom name with Tasmota module name (e.g., "Generic")
- Fix: Only set module name if controller is empty; after creation, explicitly call `renameDevice` RPC if user's name differs from DB
- Fixed yellow "no internet" banner flash on Android launch
- Root cause: Connectivity check ran immediately on `initState`, before Android network stack was ready
- Fix: Delayed initial check by 3 seconds in `home_screen.dart`
- Codemagic build: `69b7ccc51bfb39ba45b2c6ef`
- ASC build ID: `1788cac9-9568-4b57-a87a-019e8441154a`

## Patterns learned
- MQTT state streams + `showModalBottomSheet` = iOS dismissal bug. Use `_isBottomSheetOpen` flag pattern for any future bottom sheets in screens with live MQTT streams
- Tasmota device discovery overwrites text controllers — always check if user has entered custom data before auto-filling
- Android connectivity checks can fail briefly on cold start — delay initial check
