### Android build lessons learned
- This server has `/tmp` with `noexec` — always set `JAVA_OPTS=-Djava.io.tmpdir=/root/gradle-tmp`
- Flutter 3.24 Kotlin DSL projects need a Groovy init script at `/root/.gradle/init.d/flutter-compat.gradle` to create `flutter` extension for old Groovy plugins
- Don't create extension for `:app` project (Flutter plugin does that) — only for subprojects

### Tim's note on Android
- Reminded that Android has different behaviors: auto-discovery, native functionalities
- Need to audit platform-specific code paths for device discovery (mDNS/NsdManager vs Bonjour)

### Key regex migration lessons
- `AppTheme.getTextSecondary(context)` replacement removes the closing `)` — must handle `.withOpacity()` chains
- Multi-line regex replacements in Dart need careful paren/comma tracking
- Python regex > sed for multi-line patterns
- Always run `flutter analyze` after bulk replacement to catch syntax breaks
